### 🏗️ PHASE 1: The Great Linkage & Portability Purge
*The Red Team unanimously declared that MathLib is currently a "header-only library pretending to be a static library" and is completely blocked for MSVC (Yellow Team). We must fix the physical architecture.*

#### 1.1 Create the Compiler Abstraction Layer (`include/mathlib/ml_compiler.h`)
*   **Mandate:** No file in the codebase may use `__attribute__`, `__builtin_*`, `__asm__`, or `_mm256_*` directly.
*   **Action:** Create `ml_compiler.h` containing:
    *   `ML_INLINE` (`static __forceinline` for MSVC, `static inline __attribute__((always_inline))` for GCC/Clang).
    *   `ML_ALIGN(n)` (`__declspec(align(n))` vs `__attribute__((aligned(n)))`).
    *   `ML_API` (`__declspec(dllexport/dllimport)` vs `__attribute__((visibility("default")))`).
    *   `ML_RESTRICT` (`__restrict` vs `restrict`).
    *   `ML_LIKELY(x)` / `ML_UNLIKELY(x)`.
    *   `ML_TARGET_AVX2` (GCC target attribute vs empty for MSVC).
    *   `ML_CTOR` (Constructor attribute vs empty/pragma for MSVC).
    *   `ML_DEPRECATED(msg)`.
    *   `extern "C"` guards for C++ consumers.

#### 1.2 Eradicate Global State & Thread-Safety Breaches
*   **Mandate:** The `DESIGN_CONTRACT.md` claims "No Global State." `cpu_dispatch.c` violates this with a lazy-init race condition.
*   **Action:**
    *   Delete `g_ml_cpu_caps` and `g_ml_matmul`.
    *   Delete `ml_init_cpu_dispatch()` and the `__attribute__((constructor))`.
    *   **Refactor `ml_matmul`:** Shift to **Compile-Time Dispatch** for v11S. Use `#if ML_COMPILE_TIME_AVX2` inside `ml_matmul` to route to `ml_matmul_avx2` or `ml_matmul_scalar`. Runtime dispatch is deferred to v12.
    *   Create `src/cpu_caps.c` with pure query functions (`ml_cpu_has_avx2()`) that return compile-time constants.

#### 1.3 The Linkage Model Surgery (Headers vs. Sources)
*   **Mandate:** Stop putting 19th-degree polynomials and LU decompositions in headers as `static inline`. It causes binary bloat, I-cache pressure, and prevents patching.
*   **Action:**
    *   **Keep Inline (Trivial only):** `ml_fabs`, `ml_isnan`, `ml_isinf`, `ml_copysign`, `ml_fp_classify`, `ml_ldexp_pure`, `ml_frexp_pure`, `cplx_add`, `cplx_sub`, `cplx_mul`, `ML_TENSOR_AT`, `ml_workspace_alloc`.

        *   `src/trig.c` (all sin, cos, tan, atan, asin, acos)
        *   `src/exp_log.c` (exp, log, pow, hyperbolics)
        *   `src/linalg.c` (LU decomp, solve)
        *   `src/fft.c` (fft_execute, ifft_execute)

    *   **Delete `src/mathlib.c`**: It is currently just a list of `#include`s. It is no longer needed.

#### 1.4 Strict Aliasing & Type System Standardization
*   **Mandate:** Union type-punning (`ml_fp_cast`) is technically UB in strict C99 and MSVC can optimize it unpredictably.
*   **Action:**

### 🧮 PHASE 3: Algorithmic & Numerical Surgery (The P0 Math Bugs)
*The Red Team found silent data corruption, stack overflows, and IEEE-754 violations.*

#### 3.1 The `ml_ldexp_pure` & `exponential_ieee754` Catastrophe
*   **Bug:** `cast.u += ((uint64_t)exp << 52);` corrupts bits if `exp` is negative, because casting a negative `int` to `uint64_t` creates a massive number.
*   **Fix:** Extract exponent bits, add `exp` as a signed integer, check for overflow/underflow, and re-pack the bits.

#### 3.2 `gamma_new` Stack Overflow & `integral_traditional` Silent Failure
*   **Bug:** `gamma_new` recurses infinitely for large `x`. `integral_traditional` silently truncates after 10M steps.
*   **Fix:**
    *   Convert `gamma_new` tail-recursion to an iterative `while` loop. Add an overflow guard (`if (x > 171.0) return INF;`).
    *   Make `integral_traditional` return `NaN` (`0.0/0.0`) if the `max_steps` ceiling is hit, signaling failure instead of returning a wrong answer.

#### 3.3 `ml_fmod` Integer Overflow & `ml_tan`/`ml_atan2` Zero-Ignorance
*   **Bug:** `ml_fmod` caps quotient at `9.22e18` (fragile). `ml_tan` returns `-Inf` for `0/0`. `ml_atan2` ignores signed zeros (`+0.0` vs `-0.0`).
*   **Fix:**
    *   Rewrite `ml_fmod` to use iterative subtraction for pathological quotients, avoiding the `long long` cast limit.
    *   Fix `ml_tan`: `if (c == 0.0) return (s == 0.0) ? NaN : ((s > 0) ? INF : -INF);`
    *   Fix `ml_atan2`: Use `ml_copysign` and explicit `+0.0`/`-0.0` checks to match IEEE-754 Annex F.

#### 3.4 `ncr`/`npr` Intermediate Overflow & `cplx_div` Safety
*   **Bug:** `ncr` multiplies `factorial(n-r) * factorial(r)`, overflowing `uint64_t` before division. `cplx_div` overflows on large complex numbers.
*   **Fix:**
    *   Rewrite `ml_ncr` using multiplicative cancellation (e.g., `n * (n-1) ... / r!`).
    *   Implement **Smith's Method** in `ml_cplx_div` to scale real/imaginary parts and prevent intermediate `denom` overflow/underflow.

#### 3.5 LU Decomposition Singularity Drift
*   **Bug:** `if (max_val == 0.0)` misses near-singular matrices that cause NaN cascades.
*   **Fix:** Calculate a relative machine tolerance (`matrix_norm * DBL_EPSILON`) and fail loudly with `ML_ERR_SINGULAR` if the pivot drops below it.

#### 3.6 Branchless SIMD Prep
*   **Bug:** `if (m < 0.707...)` in `ml_log` causes SIMD execution divergence.
*   **Fix:** Refactor range adjustments into branchless mask evaluations using bitwise operators and `memcpy`.
