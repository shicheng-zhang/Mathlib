This is non-standard. The convention is:

# test_oracle.c

This is the most important new file in v11S.

```c
static int64_t ulp_distance(double a, double b) {
    int64_t ia, ib;
    memcpy(&ia, &a, sizeof(int64_t));
    memcpy(&ib, &b, sizeof(int64_t));
    if (ia < 0) ia = 0x8000000000000000LL - ia;
    if (ib < 0) ib = 0x8000000000000000LL - ib;
    int64_t diff = ia - ib;
    return diff < 0 ? -diff : diff;
}
```

`memcpy` for type-punning is correct and well-defined in C99. ✓

`0x8000000000000000LL - ia` when `ia == 0x8000000000000000LL` (i.e., -0.0) gives 0. ✓

But: `ia - ib` is signed integer subtraction. If `ia` and `ib` are far apart (e.g., +1.0 and -1.0), this overflows `int64_t`, which is undefined behavior in C99. In practice, for the ULP ≤ 5 threshold you're checking, this never triggers. But it's technically UB. Fix with unsigned arithmetic:

```c
static uint64_t ulp_distance(double a, double b) {
    uint64_t ia, ib;
    memcpy(&ia, &a, sizeof(uint64_t));
    memcpy(&ib, &b, sizeof(uint64_t));
    if (ia >> 63) ia = 0x8000000000000000ULL - ia;
    if (ib >> 63) ib = 0x8000000000000000ULL - ib;
    return ia > ib ? ia - ib : ib - ia;
}
```
